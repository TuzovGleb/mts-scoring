# Скоринг проекта → deep-research метапромт

Сайт-воронка оценки проектов. Автоматизирует ручной процесс из папки `МТС`:
**опросник V0 → готовый deep-research метапромт → (Фаза 2) рассылка по моделям → сравнение ответов.**

Основан на:
- `Опросник_V0_данные_компании.md` — 23 вопроса, теги Ф/О/Д, веса факторов;
- `Промт_deep-research_универсальный_v4.md` — шаблон метапромта;
- `Методология_скоринга_единая.md` — балл и уверенность.

## Статус

- **Фаза 1 (готово, релизим):** опросник + детерминированная сборка метапромта + опциональный скоринг v0. Чистая статика, без бэкенда и ключей.
- **Фаза 2 (готово, нужны ключи):** Node/Express бэкенд рассылает метапромт параллельно в Perplexity, OpenAI, Gemini, Anthropic и собирает ответы для сравнения на фронте. Протестировано end-to-end в мок-режиме; для боя впиши ключи в `.env`.

## Структура

```
scoring-app/
  frontend/            ← Фаза 1, статичный сайт без сборки
    index.html
    styles.css
    app.js             — состояние, шаги, localStorage, экспорт/импорт
    questionnaire.js   — схема опросника V0 + доп. поля
    prompt-template.js — шаблон v4 + buildPrompt() + условные секции
    scoring.js         — балл 0–100 + уверенность H/M/L
  backend/             ← Фаза 2 (готово)
    server.js          — Express: POST /api/research, GET /api/research/:id, jobs
    providers/         — perplexity / openai / gemini / anthropic (единый интерфейс)
    .env.example       — ключи и модель-id
  data/                — сохранённые сабмиты и ответы моделей (<проект>_<модель>_<дата>.md)
```

## Запуск локально (Фаза 1)

Это статика на ES-модулях — нужен любой http-сервер (через `file://` модули не грузятся).

```powershell
cd scoring-app/frontend
npx serve .
# или: python -m http.server 8080
```

Открыть выданный адрес (напр. http://localhost:3000). Прогресс автосохраняется в браузере (localStorage). Кнопки **Экспорт/Импорт** переносят черновик между устройствами.

## Деплой (Фаза 1)

Папка `frontend/` — статика без секретов, кладётся на любой статик-хостинг:

- **Vercel/Netlify:** перетащить папку `frontend/` или подключить репозиторий (root = `frontend`).
- **GitHub Pages:** запушить `frontend/` в ветку, включить Pages.

Бэкенд и API-ключи в этот деплой НЕ входят — они появятся отдельным сервисом в Фазе 2.

## Как пользоваться

1. Пройти опросник по блокам (0 → A…G → Параметры ресёрча). К содержательным ответам ставить тег **Ф/О/Д**.
2. На шаге «Результат» — готовый метапромт: **Копировать** / **Скачать .md** / открыть Perplexity/ChatGPT/Gemini/Claude вручную.
3. Опционально — проставить баллы 1–5 по факторам и получить балл v0 (0–100) с уверенностью.

## Логика сборки метапромта

`buildPrompt(answers)` подставляет ответы в шаблон v4 и включает/выключает секции:
- **тип бизнеса = маркетплейс** → добавляется Задача 3 (сторона предложения) и маркетплейс-оговорка в правиле 8;
- **гео-экспансия ≠ «нет»** → во всех релевантных задачах добавляются ветки по гео экспансии (рынок, конкуренты, риски, отдельный балл защищаемости).

Маппинг ответ → плейсхолдер описан в комментариях `prompt-template.js`.

## Фаза 2 — рассылка по моделям

### Запуск бэкенда

```powershell
cd scoring-app/backend
npm install
copy .env.example .env   # впиши ключи (без ключа провайдер отвечает заглушкой)
npm start                # http://localhost:8787
```

### Эндпоинты

- `GET /api/health` — статус провайдеров (есть ли ключ).
- `POST /api/research { prompt, models[], project }` → `{ jobId, models }`. Запускает провайдеры **параллельно**, ошибка одного не валит остальных.
- `GET /api/research/:jobId` → `{ status, results: { <модель>: { status, text, sources, error } } }`. Фронт поллит каждые 2.5 с.

Каждый ответ сохраняется в `data/<проект>_<модель>_<дата>.md` (с источниками).

### На фронте

На шаге «Результат» — карточка **«Рассылка по моделям»**: URL бэкенда, выбор моделей, кнопка «Разослать», колонки со статусом/текстом/источниками каждой модели.

### Провайдеры (`backend/providers/`)

Единый интерфейс `run(prompt) → {text, sources, raw}` + `enabled()` по наличию ключа:
- `anthropic.js` — Claude (`claude-opus-4-8`) + серверный `web_search_20260209`, стрим, обработка `pause_turn`.
- `perplexity.js` — Sonar (`sonar-deep-research`), chat/completions.
- `openai.js` — Responses API + tool `web_search`.
- `gemini.js` — `generateContent` + `google_search` grounding.

> ⚠️ Модель-id у OpenAI/Perplexity/Gemini дрейфуют — заданы через `.env` (`OPENAI_MODEL`, `PERPLEXITY_MODEL`, `GEMINI_MODEL`). Сверь актуальные id при подключении ключей. Anthropic-параметры сверены через skill `claude-api`.

### Хостинг

Бэкенд кладётся на Render/Railway/VPS; на фронте в поле «URL бэкенда» укажи его адрес.
